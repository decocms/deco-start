# deco-cx/apps Architecture

Reference for the `deco-cx/apps` monorepo ‚Äî the canonical library of Deco integrations.

## The App Pattern

Every app follows the same structure:

```
{app-name}/
‚îú‚îÄ‚îÄ mod.ts               # App factory ‚Äî exports Props, AppContext, state
‚îú‚îÄ‚îÄ manifest.gen.ts      # Auto-generated ‚Äî registers all blocks
‚îú‚îÄ‚îÄ runtime.ts           # Client-side invoke proxy (optional)
‚îú‚îÄ‚îÄ middleware.ts         # Request middleware (optional)
‚îú‚îÄ‚îÄ actions/             # Write operations (mutations)
‚îú‚îÄ‚îÄ loaders/             # Read operations (data fetching)
‚îú‚îÄ‚îÄ hooks/               # Client-side Preact hooks (optional)
‚îú‚îÄ‚îÄ sections/            # CMS-renderable UI sections (optional)
‚îú‚îÄ‚îÄ handlers/            # HTTP request handlers (optional)
‚îú‚îÄ‚îÄ components/          # Shared Preact components (optional)
‚îú‚îÄ‚îÄ utils/               # Internal utilities and types
‚îÇ   ‚îú‚îÄ‚îÄ types.ts         # API response/request types
‚îÇ   ‚îú‚îÄ‚îÄ client.ts        # Typed HTTP client definitions
‚îÇ   ‚îú‚îÄ‚îÄ transform.ts     # API ‚Üí schema.org mapping (commerce apps)
‚îÇ   ‚îî‚îÄ‚îÄ openapi/         # Auto-generated OpenAPI types (optional)
‚îú‚îÄ‚îÄ workflows/           # Background workflow definitions (optional)
‚îî‚îÄ‚îÄ preview/             # Admin preview UI (optional)
```

### mod.ts Pattern

```typescript
import manifest, { Manifest } from "./manifest.gen.ts";
import { type App, type AppContext as AC } from "@deco/deco";

export type AppContext = AC<ReturnType<typeof MyApp>>;

export interface Props {
  account: string;
  apiKey?: Secret;
  // ...
}

export default function MyApp(props: Props) {
  const state = { /* clients, config */ };
  const app: App<Manifest, typeof state> = { manifest, state };
  return app;
}
```

### manifest.gen.ts

Auto-generated by Deco. Registers all actions, loaders, handlers, sections, workflows as blocks.

### runtime.ts

```typescript
import { Manifest } from "./manifest.gen.ts";
import { proxy } from "@deco/deco/web";
export const invoke = proxy<Manifest>();
```

## Website App (`/website/`)

Base app that all storefronts use. Handles routing, SEO, analytics, image optimization, and theme.

Key areas:
- `handlers/` ‚Äî Fresh router, proxy, redirect, sitemap
- `loaders/` ‚Äî Pages, fonts, images, redirects, secrets, environment
- `sections/` ‚Äî Analytics (GA, GTM, Pixel), Rendering, SEO
- `matchers/` ‚Äî Audience targeting (device, cookie, date, location, etc.)
- `flags/` ‚Äî A/B testing, multivariate, audience segmentation

## Relationship: apps ‚Üí apps-start

```
deco-cx/apps (Fresh/Deno)          @decocms/apps-start (TanStack/Node)
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ           ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
vtex/utils/transform.ts      ‚Üí    vtex/utils/transform.ts (ported)
vtex/utils/types.ts          ‚Üí    vtex/utils/types.ts (ported)
vtex/loaders/**              ‚Üí    vtex/loaders/** + vtex/inline-loaders/**
vtex/actions/**              ‚Üí    vtex/actions/** (consolidated)
vtex/hooks/**                ‚Üí    vtex/hooks/** (React + TanStack Query)
commerce/types.ts            ‚Üí    commerce/types/commerce.ts
commerce/utils/**            ‚Üí    commerce/utils/** + commerce/sdk/**
utils/fetch.ts               ‚Üí    Replaced by vtexFetch/vtexFetchWithCookies
utils/http.ts                ‚Üí    Not needed (no OpenAPI codegen in apps-start)
utils/graphql.ts             ‚Üí    vtexIOGraphQL in client.ts
```

Key differences:
- apps-start has no `mod.ts` / `manifest.gen.ts` / `runtime.ts` (no Deco framework)
- apps-start uses `configureVtex()` instead of app factory pattern
- apps-start loaders are pure async functions, not Deco blocks
- apps-start hooks use `@tanstack/react-query` instead of `@preact/signals`
- apps-start has no OpenAPI codegen ‚Äî uses manual `vtexFetch` calls
